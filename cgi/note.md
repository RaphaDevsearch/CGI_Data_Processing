Goal
-----------
## 1. Understand how the Web works ⭐⭐⭐⭐⭐

This is the primary goal.

You learn what happens from the moment a user clicks a button until the browser displays a new page.

```
User
   │
   ▼
Browser
   │
HTTP Request
   │
   ▼
Apache
   │
Execute CGI Program
   │
   ▼
C Program
   │
Read / Write Data
   │
Generate HTML
   │
   ▼
Apache
   │
HTTP Response
   │
   ▼
Browser
```

This is the complete request-response cycle.

---

## 2. Understand HTTP

For example:

```
GET /cgi/view_data.cgi
```

or

```
POST /cgi/add_data.cgi
```

You learn:

* GET
* POST
* HTTP headers
* Request body
* Response body
* Status codes (later)

---

## 3. Understand data flow ⭐⭐⭐⭐⭐

This is probably the most important concept.

Example:

```
HTML Form
        │
        ▼
username=raphael
note=Hello
        │
        ▼
stdin
        │
        ▼
C Program
        │
Parse Data
        │
        ▼
CSV File
        │
Read Again
        │
        ▼
Generate HTML
        │
        ▼
Browser
```

You see exactly where the data goes at every step.

---

## 4. Understand persistent storage

You'll learn the difference between:

```
Memory (RAM)
```

and

```
CSV File (Disk)
```

Example:

```
char name[100];
```

exists only while the program is running.

But

```
users.csv
```

remains after the program exits.

---

## 5. Understand backend programming

You'll build features like:

* Register
* Login
* Add Data
* View Data
* Update Data
* Delete Data

These are the same backend operations found in larger frameworks.

---

## 6. Understand dynamic HTML

Instead of serving a static page:

```
index.html
```

your CGI program generates HTML on demand:

```c
printf("<h1>%s</h1>", username);
```

The browser can't tell whether the HTML came from a file or was generated by C.

---

## 7. Understand CRUD

Every web application performs these operations:

```
Create
Read
Update
Delete
```

Examples:

```
Create  -> Add Data
Read    -> View Data
Update  -> Edit Data
Delete  -> Remove Data
```

---

## 8. Prepare for modern frameworks

Once you understand CGI, you'll recognize the same architecture in frameworks like Laravel, Django, Express.js, Flask, Spring Boot, ASP.NET, or Next.js.

For example:

```
Browser
     │
     ▼
Request
     │
     ▼
Backend
     │
     ▼
Database
     │
     ▼
Response
```

The technologies change, but the overall flow is the same.

---

# My recommendation

Since you're building this as a learning project, don't think of it as **"CGI Data Processing."**

Think of it as:

> **"Understanding the complete lifecycle of data in a web application."**

Every feature you implement should help answer one question:

> **"Where is the data now?"**

For example, when a user clicks **Add Data**, you should be able to trace the data through every stage:

```
User types "Hello"
        │
        ▼
HTML Form
        │
        ▼
HTTP POST Request
        │
        ▼
Apache
        │
        ▼
stdin
        │
        ▼
C Program
        │
        ▼
Parse POST Data
        │
        ▼
Write to CSV
        │
        ▼
Generate HTML Response
        │
        ▼
Browser displays "Data added successfully."
```
